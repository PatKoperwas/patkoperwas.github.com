strong parameters

Rails 4 makes strong parameters the default option for mass-assignment protection. It does away with some of the problems of the older attr_protected and attr_accessible which have now been removed. One of the main drawbacks of the older ways was that it became difficult and cumbersome to specify levels of accessibility. An admin may have different write privileges than a normal user. Or if you were building a multi-step wizard certain steps would or would not be allowed to modify model data.

Using attr_accessible we also introduce a hidden logic into our models. What it says is that if this hash has this particular key, assign it, else throw an error.

Strong parameters moves that logic out of the model and closer to where the data is coming from - the controller. Strong parameters allows us to whitelist permitted parameters easily. We can have different sets of permitted params depending on the state of the application or the currently signed in user.

Strong parameters is also flexible enough to be used outside of controllers. Any place that I might get model attributes I can filter them out to ensure I do not open myself up to a mass-assignment vulnerability. This could be JSON that I get from a third-party API or another gem.
